Lecture Notes

In the last lesson, you spent a considerable amount of time investigating options in Visual Studio. As you learned, the IDE provides a variety of features to the programmer, including the ability to create project solutions, debug programs, and edit form objects that the end user can interact with. In the last lesson, you also spent a good deal of time creating a form, adding a picture to that form, and then running the project to display the design as an interactive project. What you didn't do, however, was write any actual C# code to control the form in any way. In a technical sense, the last lesson had you create a programming project with very little "programming." This lesson is the exact opposite of the last lesson, in that it just focuses on programming with the C# language.

At this point in your course, the discussion of Visual C# has been primarily theoretical. You’ve learned about the different programming methods and languages and the tools used to program. These are all different, and important, ingredients used when programming, but the most important ingredient of all is the code itself! Today, you’ll dive into programming with C#. What's important to notice is that I didn't say "Visual" C#,” but rather just "C#." Why is this important to note? C# reflects the language the programmer develops in. The term Visual connotes the user's interactivity with using the program developed by the language. It's important you understand this one last concept before you continue your journey into programming with Visual C#.

Think of Visual C# as a theatrical performance, where what you see onstage and what happens behind the scenes are two mutually exclusive things. Both of these things are related, and most certainly must be in sequence with one another, but what happens onstage is only the "presentation," whereas the "logic" of the performance—what lights should be on/off, where the audio should feed from, where characters should enter and leave, or what props the characters should have available—is all maintained behind the curtain. Visual C# is the same concept, where the "presentation layer" and the "business," or “logic,” layer are two separate things. With our business layer, we can still receive feedback from the presentation layer to determine what the end user is doing, such as if he or she has entered data, pressed a button, or even closed a window. What we do with that data, however, is completely irrelevant to what the end user can see. To that point, there are some applications where a user interface isn't even required!

So as you follow the lesson today, understand that the code you will see examples of, and write with, will be a similar code that you write when you have a visual element (such as a form) in your program. What will change is how a user will interact with the program. In today's lesson, you will use the basic console application, which is just a simple command-based terminal window that can read text and have text written to it, but not much more than that visually. The capabilities for a console application, however, are endless!

A console application, in the context of C#, is an application that takes input and displays output at a command line console with access to three basic data streams: standard input, standard output and standard error.

 

A console application facilitates the reading and writing of characters from a console - either individually or as an entire line. It is the simplest form of a C# program and is typically invoked from the Windows command prompt. A console application usually exists in the form of a stand-alone executable file with minimal or no graphical user interface (GUI). (Retrieved from: https://www.techopedia.com/definition/25593/console-application-c)

C# console applications have all the features of the .NET Framework. Console applications can read/write data to text files, databases, or Web services. They can transfer files across networks, or even the Internet. They can perform a batch of commands at the press of a (keyboard) button. Do not be fooled; a console application may look bland, but its power is anything but.

So with that in mind, this lesson will introduce you to a number of concepts for programming with C#. This lesson will showcase commenting code, reserved keywords, and working with input/output of data with an end user's interaction. You will also explore data types, including the different types of integers, floating-point precision numbers, and even text-based string data types. The lesson also showcases some of the common uses of these data types, including arithmetic against number typed values, comparing two different values to one another, and how data are committed to the computer's memory for processing. Are you excited yet? If so, let’s get started!
