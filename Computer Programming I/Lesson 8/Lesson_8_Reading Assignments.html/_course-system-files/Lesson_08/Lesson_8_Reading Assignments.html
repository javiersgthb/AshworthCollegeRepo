Lecture Notes

Welcome to the final lesson! In this lesson, we take a look at building classes, controlling access to members of a class, and creating constructors. The Time class case study in your text presented a complete class declaration consisting of private data, overloaded public constructors for initialization flexibility, properties for manipulating the class’s data, and methods that returned string representations of a Time object in two different formats.

  • Method ToUniversalString takes no arguments and returns a string in universal-time format, consisting of six digits—two for the hour, two for the minute and two for the second. For example, if the time were 1:30:07 PM, method ToUniversalString would return 13:30:07. Note that “If the date and time instance value is an ambiguous time, this method assumes that it is a standard time. (An ambiguous time is one that can map either to a standard time or to a daylight saving time in the local time zone) If the date and time instance value is an invalid time, this method simply subtracts the local time from the local time zone's UTC offset to return UTC. (An invalid time is one that does not exist because of the application of daylight saving time adjustment rules.)” (Retrieved from: https://msdn.microsoft.com/en-us/library/system.datetime.touniversaltime%28v=vs.110%29.aspx)
  • Method ToString takes no arguments and returns a string in standard time format, consisting of the hour, minute and second values separated by colons and followed by an AM or PM indicator (e.g., 1:27:06 PM). Remember that every class can declare a ToString method that returns a string representation of an object of the class and that this method is invoked implicitly when an object of a class is output as a string or concatenated with a string. Also note that “ToString is virtual. It returns a string representation. We must override ToString on custom types for the method to be effective. For numeric types, there are performance and functional differences with ToString.” (Retrieved from: http://www.dotnetperls.com/tostring)

In this lesson, you will also learn that the this reference is used implicitly in a class’s non-static methods to access the class’s instance variables and other non-static methods. You are shown explicit uses of the this reference to access the class’s members (including hidden fields) and how to use keyword this in a constructor to call another constructor of the class. In Visual Studio 2017,

 

The this keyword refers to the current instance of the class and is also used as a modifier of the first parameter of an extension method.

 

The following are common uses of this:

  • To qualify members hidden by similar names, for example: 

    C#

    public Employee(string name, string alias)
    {
         // Use this to qualify the fields, name and alias:
         this.name = name;
         this.alias = alias;
    }
  • To pass an object as a parameter to other methods, for example:
  • CalcTax(this);
  • To declare indexers, for example: 

    C#

    public int this[int param]
    {
         get { return array[param]; }
         set { array[param] = value; }
    }

Static member functions, because they exist at the class level and not as part of an object, do not have a this pointer. It is an error to refer to this in a static method. (Retrieved from: https://docs.microsoft.com/en-us/previous-versions/visualstudio/visual-studio-2012/dk1507sz(v=vs.110)):

As you complete this final lesson, you will see that composition enables a class to have references to objects of other classes as members. You will learn about C#’s garbage-collection capability and how it reclaims the memory of objects that are no longer used. Every object you create uses various system resources, such as memory. The CLR performs automatic memory management by using a garbage collector to reclaim the memory occupied by objects that are no longer in use. The destructor is invoked by the garbage collector to perform termination housekeeping on an object before the garbage collector reclaims the object’s memory. Because of this, memory leaks, which are common in other languages like C and C++ (because memory is not automatically reclaimed in those languages), are less likely in C#.
